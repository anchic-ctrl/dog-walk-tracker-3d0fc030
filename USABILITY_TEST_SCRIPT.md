# 易用性測試腳本

> **測試目的**：驗證真實使用者（老闆與員工）能否在**不需要教學**的情況下順暢完成日常任務。
>
> **測試方式**：請受測者操作，旁邊觀察並記錄。**不要主動提示**，只在受測者完全卡住時才引導。

---

## 測試前準備

| 項目 | 說明 |
|---|---|
| 測試裝置 | 受測者自己的手機（最真實的場景） |
| 測試資料 | 預先建立 3-5 隻測試狗狗（含照片、不同房間） |
| 帳號準備 | 老闆帳號（super_admin）、員工帳號（staff）各一 |
| 記錄方式 | 螢幕錄影 + 旁邊紙筆記錄卡住的地方 |
| 時間 | 每人約 20-30 分鐘 |

### 觀察重點

在整個測試過程中，注意記錄以下行為：
- 😕 **猶豫** — 手指停在螢幕上不動 > 3 秒
- ❌ **誤操作** — 點錯按鈕或進錯頁面
- ❓ **發問** — 主動問「這是什麼意思？」
- 😤 **挫折** — 明顯不耐煩或放棄嘗試

## 腳本 A：老闆 / 管理者（Admin）

> 角色設定：你是寵物旅館的老闆，今天要用新系統管理住宿的狗狗和員工。

### 場景 A1：首次登入
**任務**：「請你用你的 email 登入這個系統」

| 步驟 | 觀察點 |
|---|---|
| 找到登入頁面 | 是否理解要輸入 email？ |
| 輸入 email 並送出 | 有沒有猶豫？ |
| 查收信箱並點擊連結 | 知道要去信箱嗎？能找到信件嗎？ |
| 成功進入系統 | 首頁載入後，第一反應是什麼？ |

**觀察記錄**：
- [ ] 順利完成？  花了多久？ _____
- [ ] 卡在哪裡？ _____

---

### 場景 A2：查看放風看板
**任務**：「現在請告訴我，今天有幾隻狗在散步？哪隻狗有需要特別注意的地方？」

| 步驟 | 觀察點 |
|---|---|
| 看到首頁統計數字 | 能快速找到散步中數量嗎？ |
| 辨識警告標籤 | ⚠️ 圖示和文字是否看得懂？ |

**觀察記錄**：
- [ ] 正確回答？ _____
- [ ] 有沒有不確定的地方？ _____

---

### 場景 A3：幫狗狗散步
**任務**：「請幫 [狗名] 開始散步，然後結束散步後記錄牠去大小便的狀況」

|---|---|
| 找到該狗的卡片 | 在列表中找得到嗎？ |
| 按「開始散步」 | 按鈕位置直覺嗎？ |

| 按「結束散步」 | 知道怎麼結束嗎？ |
| 填寫大小便記錄 | 彈出的編輯表單看得懂嗎？ |
| 儲存記錄 | 知道按哪裡保存嗎？ |

**觀察記錄**：
- [ ] 順利完成？ _____
- [ ] 大小便記錄的選項是否看得懂？ _____
- [ ] 有試圖取消嗎？知道取消會刪除記錄嗎？ _____

## 腳本 B：員工（Staff）

## 測試後訪談問題

完成所有任務後，問受測者以下問題：

### 共同問題
1. 「整體來說，系統好用嗎？1-5 分你會給幾分？」
2. 「有哪個功能覺得操作起來最不直覺、最卡的？」
3. 「有沒有文字或按鈕的說明你看不太懂的？」
4. 「如果只能改一件事讓它更好用，你會改什麼？」
5. 「你覺得這個系統能取代你們現在的做法嗎？有什麼缺少的功能？」

### 老闆追加問題
6. 「管理員工的流程清楚嗎？會不會怕按錯停用帳號？」
7. 「新增/編輯狗狗的表單欄位夠用嗎？有沒有需要但沒有的欄位？」

### 員工追加問題
6. 「每天上班要用這個系統記錄，你覺得會花很多時間嗎？」
7. 「有沒有哪些日常的工作，你覺得系統應該要有但目前沒有的？」
