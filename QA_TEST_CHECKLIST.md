# 上線前 QA 測試清單

> [!IMPORTANT]
> 建議在上線前完成這份測試——因為涉及 **登入流程、權限控管、資料寫入刪除**，一旦上線出問題影響範圍大，事後修復也更複雜。

## 測試前準備

| 項目 | 說明 |
|---|---|
| 測試帳號 | 準備 3 組 email：一個 super_admin、一個 admin、一個 staff |
| 測試資料 | 至少 2 隻狗（不同房間顏色、不同體型） |
| 測試裝置 | 手機瀏覽器（主要使用場景） + 桌機瀏覽器 |
| 測試環境 | 部署到 staging 環境（或用 production Supabase + Vercel preview） |

## 1. 登入與驗證 🔐

- [ ] **1.1** 輸入**已邀請** email → 收到 magic link → 點擊連結成功登入
- [ ] **1.2** 輸入**未邀請** email → 顯示「此 Email 尚未被邀請加入系統」
- [ ] **1.3** 輸入**已停用** email → 顯示「帳號已停用」
- [ ] **1.4** 輸入空白 / 非 email 格式 → 瀏覽器阻擋送出（`type="email"` + `required`）
- [ ] **1.5** Magic link 過期後點擊 → 顯示「登入連結已失效」 
- [ ] **1.6** 重複點擊同一 magic link → 不會報不明確錯誤
- [ ] **1.7** 登入後刷新頁面 → 維持登入狀態（session persist）
- [ ] **1.8** 點擊登出 → 回到登入頁，無法直接訪問受保護路由

## 2. 邀請與接受邀請 📬

- [ ] **2.1** Admin 邀請新 staff → members 表新增一筆 `status='invited'` 記錄
- [ ] **2.2** 被邀請者收到 email → 點擊連結進入 `/accept-invite` 頁面
- [ ] **2.3** 成功接受邀請 → 帳號變為 `status='active'`，可正常使用系統
- [ ] **2.4** 重送邀請 → 被邀請者收到新的 magic link（舊的失效）
- [ ] **2.5** 超過邀請有效期限（7 天）→ 顯示邀請已過期
- [ ] **2.6** 短時間內重複發送 → 顯示「發送頻率過高」

## 3. 放風看板（首頁）🐕

- [ ] **3.2** 日期顯示正確（格式：`2026年3月4日 (二)`）
- [ ] **3.3** 散步中 / 放風中的數量統計正確
- [ ] **3.4** 有警告標籤的狗（單牽、對狗有反應、尿布墊、包尿布）顯示 ⚠️ 警告
- [ ] **3.5** 點擊「開始散步」→ 狗卡片顯示「散步中 X分鐘」，按鈕變「結束散步」
- [ ] **3.6** 散步中時「開始室內放風」按鈕為 disabled 狀態（反之亦然）
- [ ] **3.7** 「開始室內放風」→ 彈出下拉選單選擇空間（1樓客廳/2樓大房間/2樓小房間）
- [ ] **3.8** 點擊「結束散步/放風」→ 跳轉到該狗檔案頁並自動開啟記錄編輯
- [ ] **3.9** 點擊狗狗名字或照片 → 進入狗狗詳細資料頁
- [ ] **3.10** 點擊「查看詳細資料」→ 進入狗狗詳細資料頁

## 4. 狗狗詳細資料頁 📋

- [ ] **4.1** 顯示基本資訊（名字、品種、房間、體型、照片）
- [ ] **4.2** 顯示散步注意事項（單牽、對狗有反應、需用嘴套、需單獨遛、備註）
- [ ] **4.3** 顯示室內放風注意事項（尿布墊、包尿布）
- [ ] **4.4** 顯示飲食資訊、用藥資訊、其他備註
- [ ] **4.5** 顯示今日散步記錄列表（按時間排序，最舊在上）
- [ ] **4.6** 顯示今日室內放風記錄列表
- [ ] **4.7** 每筆記錄顯示：開始時間、結束時間、大便狀態、小便狀態、如廁地點、備註、紀錄者
- [ ] **4.8** 可以「開始/結束散步」和「開始/結束室內放風」
- [ ] **4.9** Admin 可以看到「編輯狗狗資料」按鈕；Staff 看不到

### 活動記錄編輯

- [ ] **4.10** 自己建立的記錄 → 可以編輯和刪除
- [ ] **4.11** 別人建立的記錄 → Staff 看不到編輯/刪除按鈕
- [ ] **4.12** Admin 可以編輯/刪除任何人的記錄
- [ ] **4.13** 編輯記錄 → 可修改開始時間、結束時間、大便狀態、小便狀態、如廁地點、備註
- [ ] **4.14** 刪除記錄 → 記錄消失，資料庫同步刪除
- [ ] **4.15** 結束活動後自動開啟記錄編輯 → 取消時刪除該筆記錄
